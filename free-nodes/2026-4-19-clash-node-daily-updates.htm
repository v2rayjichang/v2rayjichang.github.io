第一章 冲突的多棱镜：超越表象的深层动因

1.1 资源争夺战：被压缩的生存空间

1.2 文化碰撞：当差异变成鸿沟

深圳白石洲的"饺子汤圆之争"曾引发热议：北方居民晾晒腊肠的气味引发南方邻居投诉。这类文化摩擦背后，潜藏着更深层的符号冲突——不同群体对"理想生活"的想象存在根本分歧。移民城市的调查显示，68%的文化冲突源于对公共空间使用规范的认知差异。

1.3 经济压力锅：焦虑的传导机制

广州某城中村的研究揭示：每当房租上涨10%，邻里纠纷报案量就增加7.2%。经济下行期形成的"剥夺感"会扭曲人际关系，将菜市场的砍价升级为楼道里的肢体冲突。这种"压力传导效应"使得原本微不足道的摩擦演变为系统性对抗。

2.1 信任体系的崩塌

持续冲突会导致"社区记忆污染"——居民开始用怀疑眼光审视所有邻里互动。北京某社区调查显示，经历重大冲突后，居民互助行为减少43%，这种社会资本的流失需要三代人才能修复。

2.3 经济活力的窒息

FAQ

PassWall2 的节点测速结果如何影响分流？
PassWall2 会根据节点延迟和成功率自动调整策略组选择最优节点。测速结果直接影响程序选择出口，从而提高连接速度和稳定性。
SSR 如何防止流量被 DPI 检测？
通过协议插件和混淆方式可以模拟正常 HTTPS 或 HTTP 流量，隐藏真实流量特征。合理配置混淆插件和加密方式，可以有效绕过 DPI 检测，保证连接稳定性和长期可用性。
Nekoray 的内置延迟测试为什么与系统 ping 不同？
客户端的延迟测试通常基于 TCP 或特定探测包，而系统 ping 使用 ICMP，两者网络路径与优先级可能不同。此外，防火墙或服务器对 ICMP 的响应可能被限制，导致测试结果存在差异。
Potatso Lite 是否支持 Vless？
目前 Potatso Lite 仅支持 Shadowsocks 和 Vmess 协议，不支持 Vless。但开发者正在逐步扩展功能，未来可能加入更多现代代理协议支持。
Clash 的配置文件是 YAML 格式吗？
是的。Clash 的配置文件采用 YAML 格式，结构清晰易读。用户可以通过编辑 proxies、proxy-groups 和 rules 部分来自定义节点列表、分组策略和分流规则，非常灵活。
WinXray 是否支持多订阅源？
支持。用户可导入多个订阅 URL，每个订阅源独立管理。通过切换订阅源或合并策略组，可灵活管理节点，实现不同地区或用途的线路分配。
Matsuri 节点延迟排序功能如何使用？
Matsuri 会定期测速节点延迟，并根据结果排序。用户可选择延迟最低的节点进行访问，结合策略组实现自动切换，提高网络速度和连接稳定性。
Mellow 是否支持自定义 DNS？
支持。Mellow 的配置文件中可设置 DNS 服务器地址及模式（如 DoH、DoT）。用户可使用安全 DNS，如 Cloudflare 1.1.1.1 或 Google DNS，以减少污染风险。
Clash 自动节点切换如何改善网页访问速度？
Clash 结合策略组和延迟检测功能，自动选择延迟低、稳定性高的节点处理网页流量。提升访问速度和稳定性，减少加载时间和浏览卡顿，优化整体网络体验。
Vmess 是什么协议？
Vmess 是 V2Ray 的核心传输协议之一，提供高效的加密通信。它在客户端与服务器之间建立安全通道，支持多种传输方式，如 TCP、WebSocket 和 gRPC 等。由于其加密特性，Vmess 常用于绕过网络限制与防火墙检测。
